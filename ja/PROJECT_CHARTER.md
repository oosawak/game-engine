# プロジェクト憲章

## プロジェクト名

ゲームエンジン

## 目的

Web を中心に動作する、拡張性のあるゲームエンジン基盤を構築する。

クライアント側のゲーム開発、エディタ支援、将来的なネットワーク連携までを見据えた共通基盤とする。

## 背景

本プロジェクトは、次の要素を中心としたプラットフォームの基盤を提供することを目指す。

- TypeScript によるアプリケーションおよびスクリプト
- Web 描画とブラウザ実行環境のサポート
- 高負荷処理のための WASM 活用
- 低レベル処理や共有ロジックのための Rust 活用
- 将来的なリアルタイム通信のための WebRTC 活用

## 成功条件

以下を満たせばコアエンジンは成功とみなす。

- 初期化と終了が安全にできる
- 安定したゲームループを実行できる
- シーンとゲームオブジェクトを管理できる
- コンポーネントベースのライフサイクルを提供できる
- `Transform` を必須コンポーネントとして扱える
- 描画、入力、リソース管理を後から拡張できる

## スコープ

### 対象範囲

- エンジンコア
- シーン管理
- GameObject 管理
- Component ライフサイクル
- Transform と基本数学
- TypeScript 向けの初期 API 設計

### MVP では対象外

- 完全な物理エンジン
- 高度なエディタ UI
- マルチプレイヤー通信実装
- アセットパイプライン
- ECS への移行
- 本格的なレンダラー実装

## 設計指針

- コアを小さく保つ
- 責務をモジュール単位で分離する
- ライフサイクルを明示する
- Rust/WASM と自然につながる構造にする
- シーンロジックと描画詳細を強く結び付けない

## 主な成果物

- コア設計書
- TypeScript 初期ソース構成
- ライフサイクルとシーン動作の単体テスト
- 初期ゲーム開発に使える安定した API

## 利害関係者

- エンジン開発者
- ゲームプレイ開発者
- ツール/エディタ開発者
- 将来のネットワークおよびサーバー連携担当

## 前提

- 主な実行環境はブラウザ
- エンジンの統括は TypeScript で行う
- Rust/WASM の統合はコア安定後に追加する
