# ビジョン

## ビジョンステートメント

Web エコシステムで快適に動作し、ゲーム、ツール、リアルタイム連携の共通基盤として成長できる軽量かつ拡張可能なゲームエンジンを作る。

## 長期的な方向性

エンジンは以下を支える基盤へ発展させる。

- ブラウザ中心のゲーム開発
- 再利用可能なゲームロジックとエンジンロジック
- 複数の描画バックエンド
- 可能な範囲でのクライアント/サーバー間の共有ロジック
- エディタ支援ワークフロー
- WASM または Rust による高性能計算

## 望ましい特性

- すぐ始めやすい
- 理解しやすい
- 拡張しやすい
- テストしやすい
- リファクタリングしやすい
- 所有権とライフサイクルが明確

## アーキテクチャ上の性格

このエンジンはモノリシックなフレームワークではない。役割ごとに分かれたモジュール群として構成する。

- `core` はループと全体制御
- `scene` はワールド状態
- `object` は所有関係
- `component` は振る舞い
- `rendering` は描画
- `input` は入力
- `resource` はアセット
- `wasm` は高負荷処理

## 将来のプラットフォーム像

将来的には、次のような機能群とつながることを想定する。

- 共有プロトコル定義
- WebRTC ベースのリアルタイム通信
- サーバー権威型ゲーム
- アセットおよびシーンエディタ
- プラグインによる拡張

## 避けるべきこと

- ゲームロジックをエンジンコアに直書きすること
- オブジェクトロジックを描画実装に密結合させること
- ライフサイクルをモジュールごとに不一致にすること
- 単一デモ専用の構造に閉じること
